# Veil Scout

> 用积分（credits）预测，用 zkID 证明唯一性，通过发现（scout）真正的 Builder 来赚取奖励。

**开发周期：** 60 天（AI x Web3 黑客松）

---

# 1. 问题

HTX 黑客松、AI x Web3 生态、DAO、孵化器都面临同一个问题：

> **项目很多，但没有可靠的方式分辨谁真的能交付，谁只是在包装。**

传统评价方式——pitch deck、demo 演示、团队背景、故事叙述、KOL 背书、短时间评委打分、社区热度——都容易被操纵，也容易误判。

有些项目外表光鲜但没有真正的执行力。有些项目看起来粗糙、团队默默无闻，但技术扎实、真的能交付。

**Veil Scout 要解决的问题是：**

> **如何用市场激励机制，让社区中最聪明的人去发现哪些项目具有真正的执行力？**

---

# 2. 核心定位

这**不是** Polymarket 的复制品。它不预测：

```
伊朗和美国会开战吗？
BTC 会到 20 万美元吗？
某个代币会上所吗？
```

Veil Scout 预测的是：

```
这个 AI Agent 能完成 500 次经验证的链上操作吗？
这个项目能在 30 天内达到 1,000 个活跃钱包吗？
这个智能合约能通过安全审计吗？
这个团队能完成 HTX 指定的里程碑吗？
这个 Builder 能通过匿名后端挑战吗？
```

它是一个**项目发现预测市场**，不是娱乐性预测市场。

更精确地说：

> **Scout-to-Earn 预测网络** —— 用预测市场机制做项目尽调，用代币激励奖励准确的判断。

---

# 3. 积分制，不用真钱

用户不用 USDC/USDT 对 YES/NO 结果下注。他们使用平台内部的积分系统：

**Scout Credits：**
- 不可转让
- 不可提现
- 每赛季固定分配（例如每人每赛季 10,000）
- 只能用于预测市场

### 运作方式

创建一个市场：

```
AgentPay 能在 30 天内完成 1,000 笔经验证的 AI Agent 支付吗？
```

- 用户认为能 → 用 Scout Credits 买 YES
- 用户认为不能 → 用 Scout Credits 买 NO

到截止日，里程碑通过链上/可验证数据进行核实：
- 通过 → YES 赢
- 未通过 → NO 赢

预测正确 → 获得 Scout Credits + Scout Score。
预测错误 → 损失积分。

### 赛季末奖励

排行榜上表现最好的人获得：

```
代币奖励（来自赛季奖励池）
Alpha Scout SBT（链上声誉徽章）
生态白名单
HTX / B.AI / DAO 特权
未来赛季更高的积分分配额度
```

这不是 **Bet-to-Earn**，这是 **Scout-to-Earn**。

# 4. 用户为什么参与

五层动机设计：

### 4.1 排行榜竞争

有限的初始积分迫使用户进行策略性分配：
- 我最看好哪些项目？
- 我最质疑哪些项目？
- 该集中还是分散？

判断正确 → 分数上升。判断错误 → 分数下降。按 Scout Score 排名。

### 4.2 代币奖励

赛季奖励池分配示例：

```
赛季奖励池：100,000 代币
第 1 名：          15%
第 2-10 名：       35%
第 11-50 名：      30%
证据贡献者：       20%
```

### 4.3 Alpha Scout 声誉

准确的 Scout 积累可验证的链上声誉：

```
Scout_0x7a3f
准确率：78%
虚拟 PnL：+42,000
早期 Alpha 发现：6 次
证据分数：91
专长领域：AI Agent / 钱包 / DeFi 基础设施
```

权益：更高的奖励权重、更低的费用、早期市场准入、项目白名单、DAO 评审权重、生态研究员徽章。

### 4.4 影响资源分配

如果市场结果和排行榜排名影响到：
- 项目曝光度和资助排名
- 孵化入选和导师支持
- Demo Day 推荐
- 黑客松后投资观察名单

……那么 Scout 就不是在玩游戏——他们是在参与生态筛选。

### 4.5 证明 Alpha

很多 Web3 用户想要证明：

```
我比所有人都早发现了好项目。
我比 AI 更懂风险。
我在市场之前就看到了问题。
```

Veil Scout 把这种本能转化为**可验证的链上声誉**。

# 5. zkID：防女巫核心

一旦排行榜排名带有代币奖励，攻击方式就很明显：

```
1 个人 → 100 个钱包 → 100 × 10,000 Scout Credits
→ 跨市场对冲 → 刷排行榜 → 提取代币
```

仅靠钱包地址无法防止这种攻击，因为**一个人可以拥有无限个钱包**。

### 解决方案：zkID 防女巫门控

在领取 Scout Credits 之前，用户必须证明：

> **"我是一个唯一的、合格的参与者——但我不透露我的真实身份。"**

匿名，但不可无限复制。

### 两个执行点：

**1. 积分分配门控**

```
1 个 zkID = 1 个赛季分配额度
```

同一个真人、同一赛季 → 只能领取一次积分，无论控制多少个钱包。

**2. 排行榜身份**

排行榜分数绑定到 zkID 派生的匿名身份（`Scout_0x7a3f`），而非钱包地址。用户可以切换钱包，但不能复制他们的排行榜身份。

### 原理：nullifierHash

`nullifierHash` 是每个人在每个赛季的唯一匿名指纹：

```
赛季 = "HTX Genesis Scout Season 1"
用户生成 zkID 证明 → nullifierHash = 0xabc...
```

链上：

```solidity
usedNullifier[seasonId][nullifierHash] = true;
```

如果同一个人尝试用另一个钱包领取，nullifier 会显示他们已经领取过：

```
"本赛季的 Scout Credits 已领取。"
```

**实现方案：** Semaphore v4（轻量 ZKP 群组成员证明）或 Worldcoin IDKit 作为即插即用的备选方案。

# 6. AI：首席分析师，不是裁判

AI 不是裁判。AI 是**首席分析师和初始赔率生成器**。

当项目提交后，AI 读取：

```
项目描述
GitHub 活动
Demo 链接
合约地址
链上数据
里程碑描述
风险因素
```

并生成分析报告：

```
AI 初始概率：64%

看多：
- Demo 已上线
- 合约已部署
- AI × Web3 支付叙事契合
- GitHub 活跃度高

看空：
- 真实活跃用户少
- 无安全审计
- 里程碑可能过于激进
- 疑似刷量活动

建议市场赔率：
YES = 0.64
NO = 0.36
```

### 为什么这很重要

没有 AI → 市场以无价格、无讨论锚点启动。
有 AI → 用户可以立即反应：

```
AI 高估了 → 买 NO
AI 低估了 → 买 YES
```

AI 是**流动性提供者和话题发起者**，不是裁判。它的预测是一个供群体纠正的起始点。

# 7. 市场设计：预测什么

每个市场必须是一个**清晰、可结算的问题**。

**差的问题（太模糊，无法结算）：**

```
这个项目好不好？
这个团队有没有潜力？
这个 AI Agent 强不强？
```

**好的问题（具体、可验证、二元的）：**

```
AgentPay 能在 6 月 30 日前完成 1,000 笔经验证的 AI Agent 支付吗？
这个项目能在 Demo Day 之前部署一个经验证的智能合约吗？
这个 AI Agent 能完成 500 次链上任务且无未授权转账吗？
这个 Builder 能以 p95 延迟 < 100ms 通过后端挑战吗？
这个协议能在 14 天内达到 10,000 次测试网交互吗？
```

每个市场需要：
- 明确的目标
- 明确的截止日期
- 明确的数据来源
- 明确的通过/未通过标准

# 8. 端到端产品流程

### 第一步：赛季入场

用户连接钱包 → 系统提示 zkID 验证 → 生成证明 → 确认唯一性 → **发放 10,000 Scout Credits**。

### 第二步：项目提交里程碑

```
项目：AgentPay
赛道：AI × Web3 支付
里程碑：30 天内完成 1,000 笔经验证的 AI Agent 支付
验证来源：智能合约事件 + 后端验证器 + 唯一钱包过滤
截止日期：2026-06-30
```

创建市场：

```
AgentPay 能在 30 天内完成 1,000 笔经验证的 AI Agent 支付吗？
```

### 第三步：AI 设置初始赔率

```
AI 初始 YES：0.62

看多：Demo 已上线，合约已部署，支付流程清晰
看空：活跃钱包少，用户获取不确定，疑似刷量

市场开盘：YES = 0.62 / NO = 0.38
```

### 第四步：Scout 用积分预测

```
Scout A（检查了链上数据）：
  "只有 20 个真实钱包，大部分交易看起来像刷量。"
  → 买 NO

Scout B（发现了合作信息）：
  "他们已经接入了真实 API，合作方正在导流。"
  → 买 YES

Scout C 提交证据：
  → 链上事件分析显示刷量模式
  → 如果证据被验证有用，获得证据分数奖励
```

**规则：每个 Scout 在每个市场只能持有单边仓位（YES 或 NO，不能两个都持有）。** 这防止了无风险对冲。

### 第五步：里程碑结算

```
截止日到达。系统验证：
  总支付事件：1,483
  有效 Agent 支付：1,238
  唯一钱包：412
  无效交易已过滤：245
  
  目标：1,000 笔有效支付
  结果：通过 ✅
```

YES 赢。YES 持有者获得虚拟 PnL 和 Scout Score。NO 持有者损失积分。

### 第六步：排行榜评分

```
Scout Score =
  虚拟 PnL
  + 准确率奖励
  + 早期发现奖励
  + 证据分数
  - 风险惩罚（过度集中）
  - 垃圾行为惩罚
```

**早期发现奖励**是关键：
- 在 0.20 时买 YES，市场结算为通过 → **高分**（早期 Alpha 发现）
- 在 0.90 时买 YES，市场结算为通过 → **低分**（后期跟风）
- 在 0.10 时买 NO，市场结算为未通过 → **高分**（逆向洞察）

这奖励的是真正早期发现定价偏差的人，而不是共识形成后跟风的人。

### 第七步：赛季奖励

```
顶尖 Scout → 从赛季奖励池获得代币奖励
高准确率 Scout → Alpha Scout SBT
优质证据提交者 → 额外奖励
高声誉 Scout → 下赛季更高积分分配 + 白名单
```

### 飞轮效应

```
AI 设置初始赔率
      ↓
Scout 用积分纠正错误定价
      ↓
里程碑通过可验证数据结算
      ↓
准确的 Scout 获得分数 + 奖励
      ↓
排行榜吸引更多聪明的参与者
      ↓
生态获得更准确的项目发现信号
      ↓
项目资源分配优化
      ↓
更多项目想要被列入 → 更多市场 → 循环重复
```

# 9. 产品模块

## 模块 1：zkID 防女巫门控

```
验证唯一参与者（Semaphore v4 / Worldcoin IDKit）
防止多钱包刷积分
生成匿名 Scout ID（Scout_0x...）
强制每个 zkID 每赛季只能领取一次积分
```

## 模块 2：Scout 积分系统

```
每赛季发放不可转让积分
追踪每个市场的虚拟持仓
强制单边持仓规则（YES 或 NO，不能两者都有）
设置单市场积分上限（例如总积分的最多 20%）
防止 all-in 式的运气型排行榜操纵
```

## 模块 3：AI 分析师

```
接入项目材料（描述、GitHub、合约、链上数据）
生成初始概率估计
输出结构化的看多/看空分析
评估所提出的里程碑是否可客观结算
设置初始市场赔率
```

## 模块 4：预测竞技场

```
为每个项目里程碑创建 YES / NO 市场
用户用 Scout Credits 买入仓位
市场价格随交易活动调整
支持证据提交（链接、截图、链上分析）
实时显示赔率和持仓规模
```

**AMM 说明：** 生产版本可以使用 LMSR 进行数学上严谨的定价：

$$ C(q) = b \ln \left( \sum_{i \in \{YES, NO\}} \exp(q_i / b) \right) $$

黑客松 MVP 可以使用简化的恒定乘积或固定比例池来降低实现复杂度。

## 模块 5：结算预言机

```
读取链上事件 / GitHub 数据 / API 端点 / 测试结果
判定里程碑通过/未通过
将结果发布上链
支持挑战期（路线图 — V2）
```

## 模块 6：排行榜与奖励引擎

```
计算 Scout Score（PnL + 准确率 + 早期发现 + 证据 - 惩罚）
对所有 Scout 排名
从赛季奖励池分配代币奖励
为顶尖表现者铸造 Alpha Scout SBT
展示历史准确率、专长领域和过往记录
```

# 10. 为什么适合 HTX 黑客松

这不是一个通用预测市场。它是专为 HTX / AI x Web3 生态打造的**去中心化项目发现层**。

**生态想要知道的：**

```
谁真的能交付？
谁只是在包装？
谁值得继续支持？
谁应该进入孵化 / 资助 / 投资管道？
```

**Veil Scout 提供的：**

```
AI 初步筛选
社区 Scout 用积分预测
zkID 防止多账户作弊
里程碑结算验证判断
排行榜奖励最好的项目评估者
```

它将黑客松评价从：

```
评委在短时间内主观打分
```

升级为：

```
AI + 社区 + 市场 + 数据结算 = 持续发现系统
```

# 11. 与通用预测市场的对比

| 通用预测市场 | Veil Scout |
|------------|------------|
| 真金白银下注 | Scout Credits（不可转让） |
| 预测新闻、体育、政治、代币价格 | 预测项目执行能力 |
| 用户追求刺激 | 用户竞争排行榜、代币、声誉 |
| 看起来像赌博 | 看起来像项目发现竞赛 |
| 核心是流动性 | 核心是预测准确性 |
| 钱包可以女巫攻击 | zkID 防止多账户滥用 |
| 结果与生态脱节 | 结果影响项目资源分配 |

# 12. Pitch 总结

> Veil Scout 是一个用 zkID 门控的 Scout-to-Earn 预测竞技场，专注于 AI x Web3 项目发现。用户不拿真钱下注。每个经验证的唯一参与者获得不可转让的 Scout Credits，用这些积分预测项目能否完成客观里程碑。AI 分析师设置初始概率，人类 Scout 纠正错误定价，经验证的里程碑结果结算市场。最准确的 Scout 登上排行榜，获得代币奖励、声誉徽章和生态权益。

# 13. 一句话总结

> **用 zkID 保证每个 Scout 唯一，用 AI 启动市场，用积分预测项目执行结果，用排行榜和代币奖励发现真正看得准的人。**
